# Model Routing Policy

在每次处理用户请求时，先检查是否存在可独立分流的小任务。

优先使用 `route-to-cheap-model` 的场景：

- 摘要、改写、翻译、分类、字段抽取。
- 生成 commit message、PR description、README 草稿、标题、checklist。
- 小段代码解释，前提是代码已由用户提供，不需要读取仓库。
- regex、SQL、shell 片段草稿，前提是低风险且必须由主模型复核。

不要分流：

- 读取或修改文件。
- 执行命令、跑测试、构建、调试、验证。
- 代码审查、安全判断、高风险决策。
- 需要完整仓库上下文或隐藏上下文的任务。

分流规则：

- 只发送完成小任务所需的最小文本。
- 便宜模型输出必须由主模型复核后才能给用户。
- 如果调用失败，直接由主模型完成，不要中断任务。
- 当用户正在测试分流策略时，如果判断不分流，需要简短说明原因。
