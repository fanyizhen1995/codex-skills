## Motivation

  Qwen Image/Qwen-Image-Edit/Wan2.2-I2V-A14B-Diffusers could be launched with tensor parallelism, but its non-quantized Qwen-Image DiT
  path did not actually use TP-aware parameters for the main transformer block weights. As a result, each TP
  rank still loaded a full copy of the large transformer weights, so per-device memory did not scale down with
   --tp-size and larger TP configurations could still OOM. Before the fix, Qwen-Image and Wan2.2 could not start with 8 cards Ascend 910B4. Now Qwen-Image runs well with 4 cards and Wan2.2 runs with 8 cards.

  This PR adds real non-FSDP tensor-parallel weight loading for Qwen-Image while preserving the existing
  forward tensor shapes and generation behavior.

  The issue was stated in the issue below as the third problem. The issue was closed due to inactivity.
  https://github.com/sgl-project/sglang/issues/17618

  ## Modifications

  - Converted large Qwen-Image transformer block projections from replicated linear layers to TP-aware linear
  layers:
    - attention image Q/K/V projections use ColumnParallelLinear(..., gather_output=True)
    - attention text added Q/K/V projections use ColumnParallelLinear(..., gather_output=True)
    - attention output projections use RowParallelLinear(..., input_is_parallel=False, reduce_results=True)
    - image/text modulation projections use ColumnParallelLinear(..., gather_output=True)
    - image/text MLPs use a Qwen-specific TP feed-forward implementation with column-parallel first projection
   and row-parallel output projection
  - Kept shape-sensitive top-level projections as regular nn.Linear:
    - img_in
    - txt_in
    - proj_out
  - Updated non-FSDP loading so TP-aware parameters are sharded through their weight_loader before moving the
  local shard to device. This avoids temporarily materializing full TP-shardable checkpoint tensors on each
  accelerator rank.
  - Preserved the existing non-FSDP execution path; this does not require enabling FSDP inference.
