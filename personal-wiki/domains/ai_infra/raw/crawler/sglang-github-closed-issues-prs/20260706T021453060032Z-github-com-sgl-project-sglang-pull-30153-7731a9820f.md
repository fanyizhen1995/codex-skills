# Remove `# fmt: off` from environ.py Envs class

## Motivation

The `Envs` class in `python/sglang/srt/environ.py` wrapped its entire body in `# fmt: off` / `# fmt: on` to keep the env-var registry as a compact one-line-per-entry table. The side effect is that black-normalizations are also suppressed for the whole block.

This PR drops the `# fmt: off` / `# fmt: on` directives and lets black format the class.

## Modifications

- Removed `# fmt: off` (top of `Envs`) and `# fmt: on` (before the `# EPD` section).
- Ran `black`, which:
  - normalizes operator spacing: `EnvInt(2 ** 31)` -> `EnvInt(2**31)`
  - normalizes inline-comment spacing: `EnvBool(False) # ...` -> `EnvBool(False)  # ...`
  - wraps a few declarations whose trailing comment pushes the line past 88 chars (`SGLANG_EMPTY_CACHE_INTERVAL`, `SGLANG_WARMUP_TIMEOUT`, `SGLANG_DSA_FUSE_TOPK`)
  - drops a stray double blank line

No functional/behavioral change — declarations only.
