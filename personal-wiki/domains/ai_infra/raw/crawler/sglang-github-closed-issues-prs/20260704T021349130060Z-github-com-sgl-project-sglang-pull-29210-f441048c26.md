# Fix eagle hidden dim extraction, PD wildcard host, and cuda graph

## Summary

- Consolidate eagle draft hidden-state dimension extraction into two config-only helpers in `eagle_utils` (`get_draft_input_from_target_hidden_dim` and `get_draft_recurrent_hidden_state_spec`), removing the scattered classmethods and `fc.in_features` probing that gave wrong results for some model architectures.
- Fix PD disaggregation wildcard host rewriting: map `0.0.0.0` → `127.0.0.1` and `::` → `::1` (same-family loopback) instead of `self.local_ip`, which can be a different address family than the listener.
- Use `supports_target_verify_for_draft()` instead of `is_dflash()` for the decode cuda graph draft-worker guard.
- Add `input_embeds` fallback in `EagerRunner.load_batch` for multimodal forward batches where `input_ids` is None.
- Add `register_input_embeds` flag for tc_piecewise prefill buffer registration.
- Reformat multi-context-manager with-statements to parenthesized form.
