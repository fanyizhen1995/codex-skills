## Modifications
- Replaced `torch.cuda.reset_peak_memory_stats()` / `torch.cuda.max_memory_allocated()` with `torch.get_device_module()` calls, matching the pattern used in `gpu_worker.py` and `perf_logger.py`.
- Added `--profile`, `--num-profiled-timesteps`, and `--profile-all-stages` CLI args to enable PyTorch profiler integration, with profiling automatically disabled during warmup.
- Added `--num-outputs-per-prompt` CLI arg and forwarded it to sampling params (guarded to 1 for now).
- Cast `task_type` enum to `str()` before passing to dataset.
- Added `_normalize_task_name()` to `VBenchDataset` to map `ModelTaskType` enum values (e.g., T2I) to legacy task-name strings (e.g., text-to-image), fixing ValueError when using `--dataset vbench` with image models.
- Extended _normalize_task_name() to also resolve bare enum-key strings (e.g. "T2I", "I2I") directly, not just ModelTaskType enum objects and dotted enum reprs ("ModelTaskType.T2I"). A plain key string
  previously fell through unmapped and was returned as-is, so it never resolved to its legacy task name ("text-to-image"). (Copilot review follow-up.)
- Forwarded num_inference_steps from CLI args into the per-request RequestFuncInput in VBenchDataset.__getitem__ (matching RandomDataset). Previously VBench requests ignored --num-inference-steps and always
  used the model default, making step-count sweeps and cross-run latency comparisons invalid. When unset (None), behavior is unchanged.
