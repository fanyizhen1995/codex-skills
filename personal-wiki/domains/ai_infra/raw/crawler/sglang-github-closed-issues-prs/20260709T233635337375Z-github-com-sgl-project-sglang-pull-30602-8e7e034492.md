# [Fix] Prevent silent VLM server crash when /dev/shm is exhausted during multimodal feature transport

URL: https://github.com/sgl-project/sglang/pull/30602
State: closed
Labels: run-ci
Closed at: 2026-07-09T07:44:43Z
Merged at: 2026-07-09T07:44:43Z

Under a burst of multimodal requests, `ShmPointerMMData` can exhaust `/dev/shm` mid-copy and kill the server with SIGBUS (tmpfs pages are allocated lazily after ftruncate). Reserve pages with `posix_fallocate` so exhaustion raises a catchable `OSError`, and fall back to inline (pickled) transport instead of crashing. Also fixes `hash_feature` for lists mixing wrapped and plain tensors.
