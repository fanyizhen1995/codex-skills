Root Cause Analysis
On NPU devices, host memory allocation follows a different path compared to CUDA. Specifically:

1. When device_type == "npu", the ALLOC_MEMORY path invokes alloc_with_pin_memory() to allocate host memory.
2. However, alloc_with_pin_memory() allocates memory via standard PyTorch pinned memory mechanisms, without registering the buffer in Mooncake's internal shared memory pool.
3. Later, when register_mem_pool_host() attempts to register this buffer with Mooncake Store, Mooncake's dummy_client cannot find the buffer address in any registered shared memory region (Buffer is not in any registered shared memory).
4. This causes register_buffer() to return error code -1, leading to the RuntimeError and scheduler crash.
In short: NPU host memory is allocated outside of Mooncake's managed pool, making it impossible to register for zero-copy transfers.

Proposed Fix

Introduce a new allocation function alloc_with_host_register_npu() specifically for NPU devices when using Mooncake Store. This function allocates memory directly through the HostTensorAllocator (e.g., MooncakeHostTensorAllocator), which internally manages memory within Mooncake's registered pool. It skips the pin_memory() call because:

1. Mooncake already manages the memory internally.
2. Calling pin_memory() would copy data to a new location, breaking zero-copy semantics.
```
def alloc_with_host_register_npu(
    dims,
    dtype: torch.dtype,
    device: str,
    pin_memory: bool,
    allocator: HostTensorAllocator,
) -> torch.Tensor:
    """
    Allocate tensor using PyTorch's built-in pin_memory flag.
    For MooncakeHostTensorAllocator, skip pin_memory() because Mooncake
    already manages the memory internally; calling pin_memory() would
    copy data to a new location and break zero-copy semantics.
    """
    buffer = allocator.allocate(dims, dtype=dtype, device=device)
    return buffer


ALLOC_MEMORY_FUNCS = defaultdict(
    lambda: alloc_with_host_register,
    {
        "npu": alloc_with_host_register_npu,
        "musa": alloc_with_pin_memory,
    },
)
```
