## Mechanical verification

This is a pure mechanical flag rename. The transform and an equivalence verifier are in this gist: https://gist.github.com/hnyls2002/b1820a07f3012a87afe43cc23dfa5dfa

- `transform.sh` — deterministically regenerates the full diff from a clean `main` (140 files; verified byte-identical to this PR except for `black` line-reflows).
- `verify.sh` — proves every changed file differs from its parent **only** by the token rename: 136 files byte-identical after collapsing the `-decode` suffix, and the 4 files `black` reflowed (the longer token wrapped a list literal) are AST-identical modulo the rename. No logic drift.
