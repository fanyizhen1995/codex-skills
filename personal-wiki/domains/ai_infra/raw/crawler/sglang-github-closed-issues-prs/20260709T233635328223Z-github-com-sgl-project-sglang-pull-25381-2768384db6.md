## Motivation

at the moment GLM-Image AR supported with transformers only using SGLang as a backend for GLM-Image AR stage can support all SGLang feature including parallelism, prefix cache, etc.


## Modifications

new server_option `srt-encoder-url` with adress of AR server glm-image-ar model implementation in srt
sp with T5 fixed by extra check in `adjust_pipeline_config` and specifying `num_heads` in config only GLM-Image used vision_vocab_size instead of vocab_size for lm_head and Logits Processor that used in GraphMode for buffer
also need https://github.com/sgl-project/sglang/pull/25643 or copy config file from `vision_language_encoder` folder to a `processor` folder docs for new way run model
