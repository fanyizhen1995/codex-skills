# [Spec] Extract stateless draft prepare helpers into eagle_worker_common

Move `prepare_for_draft`, `prepare_for_draft_extend`, and `duplicate_prefix_tail_to_draft_branches` out of `base_spec_worker.py` into a new stateless `eagle_worker_common` module — none of them read `self`, so they were free functions wearing a method costume. Call sites drop `self.` (4 sites in `eagle_worker_v2.py` / `multi_layer_eagle_worker_v2.py`).

## Verification (mechanical move)

The moved bodies were never retyped; the verification chain is:

- **Extraction**: function bodies were copied from `base_spec_worker.py` by line range with a script. The only declared transforms are (1) dedent by 4 (method to module function), (2) drop the `self,` parameter line, (3) hoist the function-level lazy imports to module scope.
- **Byte-fidelity check**: an independent `sed` pipeline re-derived the expected output from the `origin/main` blob and `diff`-ed it against the new file — byte-identical modulo the three declared transforms.
- **Formatting step**: pre-commit black then reflowed a few expressions (the dedent changed available line width). AST equivalence between the byte-faithful version and the committed black output was checked via `ast.dump` equality.
- **No stragglers**: `grep` confirms no `self.prepare_for_draft*` call sites remain under `speculative/`; ruff (F401/F821) is clean across the package after removing the 8 imports orphaned in `base_spec_worker.py`.
- **Comments preserved verbatim** — no comment/docstring edits ride along.

The only non-move changes, both required by the relocation itself:

- `test_eagle_worker_v2_topk1_fastpath.py` mocked `prepare_for_draft` as an instance attribute; it now patches the module-level function (same pattern as the existing `build_tree_kernel_efficient` patch in that test).
- One comment in `kv_canary/plan_input.py` referencing `EagleDraftWorkerBase.prepare_for_draft_extend` now points at `eagle_worker_common.prepare_for_draft_extend`.
