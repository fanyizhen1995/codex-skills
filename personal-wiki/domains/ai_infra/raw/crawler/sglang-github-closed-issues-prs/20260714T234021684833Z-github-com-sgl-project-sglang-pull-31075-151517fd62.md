# [PD] Fix optimistic prefill inflight-queue hangs on parked/aborted reqs

URL: https://github.com/sgl-project/sglang/pull/31075
State: closed
Labels: run-ci
Closed at: 2026-07-14T03:34:44Z
Merged at: 2026-07-14T03:34:44Z

## Problem

Optimistic prefill parks a request in the prefill inflight queue when its prefill finishes before bootstrap completes. Two paths never drove such requests to a terminal state, so PD prefill could hang / spin the inflight queue:

- A parked (`pending_bootstrap`) request that had already reached `KVPoll.WaitingForInput` was re-appended to the inflight queue indefinitely instead of finalizing bootstrap, sending its KV, and progressing to transfer. Aborted/failed parked requests were likewise never terminated.
- A request polling a genuinely unexpected state (e.g. a non-pending request seeing `KVPoll.Bootstrapping`) hit the fallthrough branch, which logged `warning_once` and re-queued the request forever.

## Fix

- Route parked requests through the existing `handle_pending_bootstrap` helper in `process_disagg_prefill_inflight_queue`, so they finalize, retry, or terminate deterministically (same state machine already used elsewhere).
- Replace the silent re-queue fallthrough with a hard `RuntimeError` on truly unexpected poll states, so the queue can no longer spin.
- Factor the transfer-failure handling into `handle_inflight_transfer_failure`.
- Add no-op `clear()` / `abort()` to the `BaseKVSender` interface (`BaseKVReceiver` already has them) so the success path can call `clear()` unconditionally instead of an `hasattr` guard.

## Test

Covered by `test/registered/disaggregation/test_disaggregation_optimistic_prefill.py` (`TestOptimisticPrefill`, `TestOptimisticPrefillFailure`), which exercise the retry/abort inflight paths.

Follow-up to #30951.
