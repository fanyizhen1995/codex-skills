# Fix TiktokenTokenizer missing num_special_tokens_to_add

URL: https://github.com/sgl-project/sglang/pull/30643
State: closed
Labels: run-ci
Closed at: 2026-07-10T12:37:31Z
Merged at: 2026-07-10T12:37:31Z

## Motivation

`TiktokenTokenizer` (used by models that ship only a tekken `tokenizer.tok.json` with no HF `tokenizer.json`, e.g. `xai-org/grok-2`) does not implement `num_special_tokens_to_add()`.

`sglang.bench_serving` with the default `--dataset-name random` (text mode) calls it in `python/sglang/benchmark/datasets/random.py`:

```python
if return_text:
    num_special_tokens = int(tokenizer.num_special_tokens_to_add())
```

so benchmarking such a model crashes with:

```
AttributeError: 'TiktokenTokenizer' object has no attribute 'num_special_tokens_to_add'
```

Fixes #30640. This is a sibling to #29853, which fixed the missing `all_special_ids` on the same class in the detokenizer path.

## Modification

Add `num_special_tokens_to_add()` to `TiktokenTokenizer` returning `0`. tekken `encode()` adds no special tokens (`add_special_tokens` is ignored and tiktoken's `encode` adds none), so no length budget needs to be reserved.

## Reproduction (CPU only)

```python
from sglang.srt.tokenizer.tiktoken_tokenizer import TiktokenTokenizer
t = TiktokenTokenizer("<path>/tokenizer.tok.json")  # e.g. xai-org/grok-2
t.num_special_tokens_to_add()   # before: AttributeError ; after: 0
```

End-to-end: `python -m sglang.bench_serving --backend sglang --model xai-org/grok-2 --dataset-name random ...` no longer raises; with a Grok-2 server started with `--tokenizer-path <...>/tokenizer.tok.json`, TTFT/ITL are captured normally.
