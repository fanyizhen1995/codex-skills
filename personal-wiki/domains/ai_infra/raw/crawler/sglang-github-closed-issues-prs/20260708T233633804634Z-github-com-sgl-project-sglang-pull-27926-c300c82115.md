## Motivation

This PR is an alternative way to achieve https://github.com/sgl-project/sglang/pull/27680.

Deep gemm FP8 einsum silently falls back to an unoptimized path if the input tensor is not contiguous. We can make the `per_token_group_quant_8bit_v2` to emit the scale layout needed by deep gemm.

This also implicitly fixes https://github.com/sgl-project/sglang/issues/29038. The previous DSV4 `wo_a` FP8 DeepGEMM path flattened `o: [T, G, D]` into `[T * G, D]` and then viewed the scales back as `[T, G, -1]`. On the SM100 FP8 einsum path, scales are consumed as per-group slabs, so flattening across `T * G` can associate a token's FP8 codes with another group's scale. This PR quantizes `o` as `[T, G, D]` and emits the scale tensor in DeepGEMM's expected outer/group-major layout, preserving the `scale[t, g]` to `o[t, g, :]` association.
