# [style] Extract init-static values in scheduler hot path

URL: https://github.com/sgl-project/sglang/pull/30707
State: closed
Labels: run-ci
Closed at: 2026-07-10T02:33:02Z
Merged at: 2026-07-10T02:33:02Z

Follows the new **"Extract init-static values at construction"** rule (#30701): when a derived value's inputs are frozen for the object's lifetime, compute it once in `__init__` and read the attribute instead of re-deriving in hot paths.

This PR covers the **scheduler + scheduler_components** subsystem. Only values re-derived **3+ times** in hot paths are cached; single-use derivations are left alone.

## Changes

**`scheduler.py`**
- `self.enable_dp_attention` — replaces reads in `__init__` (decode_offload_manager + dp_tp_group setup) and `_maybe_report_active_ranks`
- `self.enable_unified_memory` — replaces reads in `event_loop_overlap`, `run_batch`, `on_idle`

**`schedule_batch.py`**
- Hoist `server_args = get_server_args()` to the top of `prepare_for_extend` / `prepare_for_decode` / `_mamba_radix_cache_v2_req_prepare_for_extend`; replaces 7 in-method `get_server_args().mamba_*` reads (mirrors the existing `maybe_evict_swa` pattern)

**`metrics_reporter.py`**
- `self.decode_log_interval` — replaces 4 reads incl. the per-iteration log gate

## Verification
All changes are **pure attribute-read substitutions** (no logic change). Verified by count-based equivalence (1:1 mapping of every replaced read) + AST parse + ruff/black/isort/codespell all pass. Test fixture in `test_forward_pass_metrics.py` updated for the new cached attribute.
