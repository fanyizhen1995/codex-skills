# [PD][AMD]: incremental KV transfer with decode radix cache

URL: https://github.com/sgl-project/sglang/pull/26288
State: closed
Labels: amd, run-ci
Closed at: 2026-06-13T07:06:44Z
Merged at: 2026-06-13T07:06:44Z

> Depends on #25377 (`--hicache-storage-backend mori`) for the prefill-side
> HBM+DRAM+SSD configuration used in the bench below. The feature itself
> does not depend on #25377 and works with any prefill HiCache configuration.

## Motivation

When `--disaggregation-transfer-backend mori` is used in PD-disagg, prefill re-transfers every prefix on every turn. This PR extends the existing `--disaggregation-decode-enable-radix-cache` flag (nixl / mooncake) to `mori`, so prefill only sends pages beyond the prefix decode already holds. No-op when the flag is off.



## How to enable

On the decode role only, add one flag:

```
--disaggregation-transfer-backend mori
--disaggregation-decode-enable-radix-cache
```

## Reproduce

Two-node 1P1D on AMD MI355X, DeepSeek-R1 MXFP4, DP=8 / TP=8. Prefill = HBM + DRAM + SSD (HiCache + mori L3); decode = HBM_Only (mori PD transfer forces hicache off on decode).

### Prefill (node A, e.g. 10.0.0.1)

```bash
python -m sglang.launch_server \
  --model-path $MODEL_PATH \
  --enable-cache-report --enable-metrics \
  --host 0.0.0.0 --port 30000 \
  --tp-size 8 --dp-size 8 --enable-dp-attention \
  --page-size 64 \
  --moe-a2a-backend mori \
  --disaggregation-mode prefill \
  --disaggregation-transfer-backend mori \
  --disaggregation-bootstrap-port 8998 \
  --enable-hierarchical-cache \
  --hicache-size 192 \
  --hicache-write-policy write_through \
  --hicache-mem-layout page_first \
  --hicache-storage-backend mori \
  --hicache-storage-backend-extra-config '{
    "dram_capacity_bytes": 34359738368,
    "ssd_enabled": true,
    "ssd_storage_dir": "/tmp/umbp_ssd",
    "ssd_capacity_bytes": 51539607552,
    "auto_promote_on_read": true,
    "eviction_policy": "prefix_aware_lru"
  }'
```

### Decode (node B) — baseline

```bash
python -m sglang.launch_server \
  --model-path $MODEL_PATH \
  --enable-cache-report --enable-metrics \
  --host 0.0.0.0 --port 30001 \
  --tp-size 8 --dp-size 8 --enable-dp-attention \
  --page-size 64 \
  --moe-a2a-backend mori \
  --disaggregation-mode decode \
  --disaggregation-transfer-backend mori \
  --max-running-requests 64
```

### Decode (node B) — with this PR

Same as above plus:

```bash
  --disaggregation-decode-enable-radix-cache
```

### Router + workload (node B)

```bash
python -m sglang_router.launch_router \
  --pd-disaggregation \
  --prefill http://10.0.0.1:30000 8998 \
  --decode  http://localhost:30001 \
  --host 0.0.0.0 --port 8000

python benchmark/hicache/bench_multiturn.py \
  --model-path $MODEL_PATH --port 8000 \
  --num-clients 8 --max-parallel 8 --num-rounds 4 \
  --request-length 1024 --output-length 64 \
  --request-rate 10 \
  --disable-auto-run --enable-round-barrier --seed 42
```

| 0 | 2.39 s | 2.15 s |
| 1 | 2.44 s | 1.06 s |
| 2 | 1.24 s | 0.78 s |
| **Avg** | **1.73 s** | **1.37 s** |
