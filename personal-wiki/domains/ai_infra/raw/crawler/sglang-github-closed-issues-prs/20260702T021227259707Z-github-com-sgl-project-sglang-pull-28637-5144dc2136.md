# XPU: Support head_dim=80 via padding for  flash attention SYCL kernel

URL: https://github.com/sgl-project/sglang/pull/28637
State: closed
Labels: Multi-modal
Closed at: 2026-07-01T11:47:35Z
Merged at: 

The SYCL flash attention kernel in sgl-kernel-xpu only supports specific head dimensions: 64, 96, 128, 192. Models like Qwen2-VL use head_dim=80 in their vision encoders (1280 hidden_dim / 16 heads = 80), which causes runtime failures when using XPU flash attention.

This patch implements a padding workaround in VisionIntelXPUAttention:
1. Check if head_dim is in the supported list [64, 96, 128, 192]
2. If unsupported, pad q/k/v to the nearest larger supported size
3. Run flash attention with padded tensors
4. Slice output back to original head_dim

Example: head_dim=80 → pad to 96 → compute attention → slice to 80

This maintains correctness while enabling XPU flash attention for models with non-standard head dimensions.

Tested with: Qwen/Qwen2-VL-2B-Instruct (vision encoder head_dim=80)
