### Describe the bug

Hello everyone, I would like to ask my friends a question. After setting up the ` -- bathing-mode dynamic `, I turned on the online inference service. Then two scripts were written, one is the concurrent stress testing script batching_test.py, which is used to send batch requests to the SGLang graph service, simulating multiple people sending requests to the server. Another script, openai_test. py, uses the OpenAi interface to send a single request to the SGLang graph service. However, for the same prompt words, the output images are not the same, and the images generated by batching_test. py for concurrent stress testing scripts have a high probability of distorted lines or missing main characters (which is very common when the prompt words are in Chinese). The image generated by calling the OpenAi interface script openai_test. py will be much better, very fitting the prompt words. I also found that as the number of requests increases, the errors become more severe. I checked the seed and infer_steps, and they are all consistent. To prevent uncertainty, I turned off Cache DiT acceleration. And when I set ` -- batch max size 1 `, the generated image effect will be significantly better, and the result will be the same as the image generated by a single call to the OpenAI interface.
I have reviewed the document and found the document link https://docs.sglang.io/docs/sglang-diffusion/dynamic_batching#notes mention:
~~~text
Batch shape can change kernels, so singleton and dynamic outputs are not expected to be bit-exact.
~~~
I understand from the docs that "singleton and dynamic outputs are not expected to be bit-exact", but the difference here is not minor pixel-level variation — the generated image is severely distorted with missing subjects, which is far from acceptable generation quality.
Is there any way to solve the problem of poor image generation requests when batch requests are made after setting -- batch mode dynamic?

Test concurrent code ` batching_test. py `:
~~~python
import asyncio
import aiohttp
import base64
import os
import random

API_URL = "http://localhost:30000/v1/images/generations"
MODEL_NAME = "Tongyi-MAI/Z-Image-Turbo"
TOTAL_REQUESTS = 6
BATCH_SIZE = 5

SAVE_IMAGES = True
SAVE_COUNT = TOTAL_REQUESTS

OUTPUT_DIR = "imgs"
DATA_FILE = "data4.txt"
os.makedirs(OUTPUT_DIR, exist_ok=True)
success_count = 0
fail_count = 0
with open("rule4.txt",encoding="utf-8") as f:
    rule_data=f.read().strip()
def load_prompts(file_path, count):
    with open(file_path, 'r', encoding='utf-8') as f:
        prompts = [line.strip() for line in f if line.strip()]
    if not prompts:
        raise ValueError("Prompt file is empty")
    if len(prompts) >= count:
        return random.sample(prompts, count)
    print(f"[Warning] Prompt count ({len(prompts)}) is less than request count ({count}), prompts will be reused.")
    original = prompts[:]
    while len(prompts) < count:
        prompts.append(random.choice(original))
    return prompts

async def generate_image(session, request_id, prompt, semaphore):
    global success_count
    global fail_count
    async with semaphore:
        payload = {
            "model": MODEL_NAME,
            "prompt": rule_data+prompt,
            # "prompt": prompt,
            "n": 1,
            "size": "640x480",
            "response_format": "b64_json",
            "seed":42,
            "num_inference_steps":9
        }
        try:
            async with session.post(API_URL, json=payload, timeout=300) as response:
                if response.status != 200:
                    fail_count += 1
                    print(f"[Failed] Request {request_id} HTTP {response.status}")
                    return
                result = await response.json()
                success_count += 1
                if SAVE_IMAGES and request_id < SAVE_COUNT:
                    image_b64 = result["data"][0]["b64_json"]
                    image_bytes = base64.b64decode(image_b64)
                    prompt_prefix = prompt[:9]
                    with open(f"{OUTPUT_DIR}/{prompt_prefix}.png", "wb") as f:
                        f.write(image_bytes)
        except Exception as e:
            fail_count += 1
            print(f"[Error] Request {request_id}: {e}")


async def main():
    prompts = load_prompts(DATA_FILE, TOTAL_REQUESTS)
    semaphore = asyncio.Semaphore(BATCH_SIZE)
    connector = aiohttp.TCPConnector(limit=0)
    async with aiohttp.ClientSession(connector=connector) as session:

        total_batches = (TOTAL_REQUESTS + BATCH_SIZE - 1) // BATCH_SIZE
        request_counter = 0
        for batch_id in range(total_batches):
            batch_prompts = prompts[
                batch_id * BATCH_SIZE:
                (batch_id + 1) * BATCH_SIZE
            ]
            tasks = []
            for prompt in batch_prompts:
                tasks.append(
                    asyncio.create_task(
                        generate_image(
                            session,
                            request_counter,
                            prompt,
                            semaphore
                        )
                    )
                )
                request_counter += 1
            await asyncio.gather(*tasks)
            print(
                f"[Batch {batch_id+1}] "
                f"Success={success_count} Failed={fail_count}"
            )
    print("\n========== SUMMARY ==========")
    print(f"Success: {success_count}")
    print(f"Failed: {fail_count}")

if __name__ == "__main__":
    asyncio.run(main())
~~~
