# [style] Extract init-static values in tokenizer + multimodal path

URL: https://github.com/sgl-project/sglang/pull/30709
State: closed
Labels: run-ci
Closed at: 2026-07-10T02:36:35Z
Merged at: 2026-07-10T02:36:35Z

Follows the new **"Extract init-static values at construction"** rule (#30701): when a derived value's inputs are frozen for the object's lifetime, compute it once in `__init__` and read the attribute instead of re-deriving in hot paths.

This PR covers the **tokenizer manager + multimodal processors** subsystem.

## Changes

**`tokenizer_manager.py`**
- Cache `incremental_streaming_output`, `enable_lora`, `enable_trace`, `allow_auto_truncate`, `skip_tokenizer_init` in `__init__` (mirrors the existing `self.enable_metrics` precedent). Also normalizes the operand-order inconsistency in the `incremental_streaming_output` checks (`A and B` vs `B and A`).

**`qwen_vl.py`** (2 commits — refactor + isolated cleanup)
- Cache `_spatial_merge_size` / `_tokens_per_second` in `__init__`; override the base-class `spatial_merge_size` `@property` to return the cached value (the base property has no setter, so direct `self.spatial_merge_size = ...` would raise `AttributeError`). Replaces 4 direct `self.hf_config.vision_config.spatial_merge_size` bypasses + 3 `tokens_per_second` getattrs.
- Separate commit: drop the dead `getattr(self, "spatial_merge_size", 1)` fallback (`1` was unreachable — the property always resolves).

**`base_processor.py` + `ernie45_vl.py`**
- Cache `keep_mm_feature_on_device`, `disable_fast_image_processor`, `skip_tokenizer_init` in `BaseMultimodalProcessor.__init__` (next to the existing `mm_process_config` caching); replaces reads in base + the `ernie45_vl` override.

## Verification
All changes are **pure attribute-read substitutions** (no logic change). The qwen_vl cleanup is isolated in its own commit so the refactor commit stays byte-equivalent. Verified by count-based equivalence + AST parse + ruff/black/isort/codespell all pass.
