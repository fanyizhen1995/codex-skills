## Motivation

## Summary

This PR optimizes HiCache L3 hash generation by replacing the Python per-token
SHA256 update loop with bulk uint32 byte conversion.

Previously, each page hash was computed by looping over tokens in Python:

```python
for t in token_ids:
    hasher.update(t.to_bytes(4, byteorder="little", signed=False))
```
This is expensive when L3 HiCache needs to compute hashes for many pages.

Optimization summary:

- Moved HiCache page hashing from Python per-token loops to a native C++ extension.
- The C++ path reads `array("q") `token buffers directly, avoiding per-token Python `to_bytes()` calls.
- Page hash input is built in a small reusable per-page scratch buffer instead of materializing a full token byte buffer.
- EAGLE/bigram hashing is generated directly as overlapping token pairs: `[t0, t1, t1, t2, ...]`.
- Hash semantics are unchanged: same SHA256 chaining and same per-page hex outputs.
- Result: about `70x` speedup for normal tokens and `70x `speedup for EAGLE at `page_size=64`.
