# Cap diagnostic detail computation for failing tensors

URL: https://github.com/sgl-project/sglang/pull/30656
State: closed
Labels: 
Closed at: 2026-07-09T12:19:52Z
Merged at: 2026-07-09T12:19:52Z

Percentile computation dominates per-tensor comparator cost (a
numpy.percentile call is roughly 10-40x the rel_diff reduction) while being
purely diagnostic: percentiles and samples feed the formatter output and
never affect the pass/fail verdict. When most tensors fail (e.g. a
systematic divergence between two dumps), the comparator spent almost all
of its time producing detail nobody reads past the first few examples.

Introduce FailureDisplayBudget, a small mutable budget object (max_detail=50;
negative disables the cap) that the entrypoint creates once per run and
threads through the bundle-comparison path, and restructure
compare_tensor_pair into two passes:

- Pass 1 computes the diff without percentiles and decides pass/fail.
- Only failing (or shape-mismatched) tensors that obtain a budget unit
  recompute the diff and tensor stats with full percentile detail. Passing
  tensors and over-budget failures skip percentile computation entirely.
- Callers that pass no budget (direct compare_tensor_pair use, the
  replicated-weight check) keep full detail for every failure.

compute_tensor_info / _compute_tensor_stats / compute_diff gain an
include_percentiles flag.

Pass/fail results, rel/max_abs/mean_abs values, and sample emission are
unchanged; the only observable difference is that percentile detail is
omitted for passing tensors and for failures beyond the budget.

Tests cover the budget object (exact grant count, negative-cap unlimited),
no-budget full detail, passing tensors consuming nothing, within/over-budget
failing tensors, shape-mismatch accounting, sample independence, downcast-diff
detail, the include_percentiles flags, and an end-to-end run() check.
