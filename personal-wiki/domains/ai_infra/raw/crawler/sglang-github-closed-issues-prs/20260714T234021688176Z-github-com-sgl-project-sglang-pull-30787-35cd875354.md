## Motivation

Phase 2.5 sweep **3 of 7** (see the migration plan in RFC #29630): the ~40 kernels living in modules directly under `srt/layers`, plus the `rotary_embedding` and `layers/utils` strays.

## Modifications

**Wholesale moves** (byte-identical; imports rewritten repo-wide):

| From (srt/layers) | To |
|---|---|
| `elementwise.py` (7 Triton) | `ops/layernorm/elementwise.py` |
| `gemma4_fused_ops.py` (6 Triton) | `ops/layernorm/gemma4_fused_ops.py` |
| `mhc.py` (TileLang) / `mhc_head.py` | `ops/layernorm/` |
| `deepseek_v4_rope.py` (5 Triton + TileLang pass-configs) | `ops/attention/deepseek_v4_rope.py` |
| `fused_qk_norm_rope_store.py` / `fused_qk_rmsnorm_rope_gate.py` / `fused_qk_norm.py` | `ops/attention/` |
| `rotary_embedding/triton_kernels.py` | `ops/attention/rotary_triton.py` |
| `multimodal.py` (GPU tensor hashing) | `ops/memory/gpu_tensor_hash.py` |
| `layers/utils/hash.py` (murmur3; sampler-only user) | `ops/sampling/murmur_hash.py` |

**Extractions from mixed modules** (kernel code verbatim; classes stay in srt):

- `rotary_embedding/mrope.py`: `apply_interleaved_rope_kernel` + Triton launcher → `ops/attention/mrope.py`; the `MRotaryEmbedding` family stays in srt.
- `dp_attention.py` (94 importers — only the embedded kernel moves): `memcpy_triton_kernel` + launcher → `ops/memory/memcpy_triton.py`.

Also registered the migrated public entry points as `KernelSpec` inventory.
