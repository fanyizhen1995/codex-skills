# [Fix]: Defer DSA MLA CP KV gather for fp8 trtllm prefill in PD mode

URL: https://github.com/sgl-project/sglang/pull/29161
State: closed
Labels: run-ci
Closed at: 2026-07-01T07:20:12Z
Merged at: 2026-07-01T07:20:12Z

## Summary

Fix a shape mismatch crash when running DSA prefill with context parallel (CP), fp8 KV cache, and the `trtllm` backend — a configuration commonly used in PD-disaggregated prefill serving.

When CP is enabled, each rank only holds a local token shard, but the original code still all-gathered KV in `forward_absorb_prepare` **before** fused RoPE+quantization (Has skipped RoPE). For the fp8 `trtllm` path, RoPE is intentionally deferred to `dsa_backend._forward_trtllm` via `mla_quantize_and_rope_for_fp8`. This left `k_rope` with a global batch dimension while `forward_batch.positions` only covered the local CP shard (e.g. 4 vs 1), causing `mla_rope_quantize_fp8` to fail.

This PR defers CP KV all-gather for the fp8+trtllm+CP path and performs it **after** fused RoPE quantization. The bf16 CP path is unchanged.

## Root Cause

For fp8 + `trtllm`, `_fuse_rope_for_trtllm_mla()` skips RoPE in `forward_absorb_prepare`, but `rebuild_cp_kv_cache()` was still executed under CP. 


## Changes

python/sglang/srt/models/deepseek_common/attention_forward_methods/forward_mla.py
- Add `_should_defer_dsa_cp_kv_gather()` to detect the fp8+trtllm+DSA-prefill-CP case (`dsa_prefill_cp && fuse_rope_for_trtllm_mla`).
- Cache `fuse_rope_for_trtllm_mla` once in `forward_absorb_prepare` instead of calling `_fuse_rope_for_trtllm_mla()` multiple times.
- Gate `rebuild_cp_kv_cache()` with `not defer_kv_gather_until_after_rope`, so fp8+trtllm+CP skips the early CP KV gather.
- **Unchanged:** bf16 CP, `flashmla_sparse` CP, and non-CP paths still gather KV in `forward_absorb_prepare` after local RoPE.


python/sglang/srt/layers/attention/dsa_backend.py
- Add `_all_gather_dsa_trtllm_fp8_kv()` using `cp_all_gather_rerange_output` to all-gather concatenated `k` and `k_rope` across CP ranks.
- In `_forward_trtllm()`, after `mla_quantize_and_rope_for_fp8()` and before writing KV cache, call the new helper when `save_kv_cache && dsa_use_prefill_cp(forward_batch)`.
