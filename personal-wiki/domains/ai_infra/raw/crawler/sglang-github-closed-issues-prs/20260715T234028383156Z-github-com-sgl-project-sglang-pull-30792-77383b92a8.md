## Motivation

Phase 2.5 sweep **5 of 7** (migration plan in RFC #29630): the DeepSeek DSA and DSV4 attention kernel subtrees (~48 Triton kernels + two TileLang suites).

## Modifications

**Wholesale moves** (byte-identical; imports rewritten repo-wide) into new `ops/attention/dsa/` and `ops/attention/dsv4/` subpackages:

- `dsa/`: `dequant_k_cache`, `quant_k_cache`, `triton_kernel`, `triton_sparse_mla`, `transform_index`, `tilelang_kernel` (2.5k-line TileLang suite), `index_buf_accessor` (kernels + `GetK/GetS/...` launch wrappers)
- `dsv4/`: `fused_compress_triton` (9 kernels), `fp4_indexer`, `dequant/quant_k_cache`, `metadata_kernel`, `tilelang_kernel`, `index_buf_accessor`, and the whole `unified_kv_kernels/` package (9 kernels)

**Extractions from mixed modules** (kernel code verbatim; backend/indexer logic stays in srt):

| Source (srt, stays) | Extracted kernel module |
|---|---|
| `dsa/utils.py` (CP policy helpers) | `dsa/cp_split.py` |
| `dsv4/sparse_prefill_utils.py` (workspace/cache classes) | `dsv4/sparse_prefill_kernels.py` |
| `dsv4/compress_hip.py` (`CompressorHip` backend) | `dsv4/rms_normalize_hip.py` |
| `dsv4/indexer.py` (`C4IndexerBackendMixin`) | `dsv4/fused_scale.py` |
| `dsv4/compressor_v2.py` (`CompressorBackendMixin`) | `dsv4/compress_c128_hip.py` (HIP-guarded c128 kernels) |

Registered migrated entry points as `KernelSpec` inventory.

## Verification

- Import-purity intact; `test_kernels_namespace.py` + `test_fused_op.py`: 33 passed; all moved/edited files py_compile + ruff clean.
- DSA/DSV4 correctness tests (`test/registered/attention/unittests/dsv4/`, dsa suites) exercise these kernels through the rewritten imports in their existing B200/H200 lanes.

Part of the Phase 2.5 series: #30784 (1/7), #30786 (2/7), #30787 (3/7), #30789 (4/7). Note: shares `ops/attention/__init__.py` append with 3/7 and 4/7 — trivial append-append conflict, will rebase whichever lands second.

## Kernel-migration verification (no perf / config regression)

All 19 moved DSA/DSV4 kernels are byte-identical relocations (git rename **R100** for 17; the 2 non-R100 deltas are **import-path-only**, no kernel-logic change), so latency is unchanged by construction. Import fix: `index_buf_accessor` (and siblings) were still imported from the pre-migration `srt.layers.attention.dsa` path → repointed to `sglang.kernels.ops.attention.{dsa,dsv4}.*`. No tuning configs moved; no `__file__`-relative config lookup in moved kernels → no config regression.

**Empirical A/B (H200, DSA `act_quant`, K=7168, block=128, bf16):**

| M | main µs | PR µs | Δ |
|--:|--:|--:|--:|
| 4096 | 27.77 | 26.90 | −3.1% |
| 16384 | 96.03 | 96.01 | −0.0% |

→ within noise; no latency regression.
