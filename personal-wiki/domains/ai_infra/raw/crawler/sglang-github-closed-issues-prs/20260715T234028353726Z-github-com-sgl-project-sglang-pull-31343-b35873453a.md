# Fix MiMo-V2 on Blackwell: FA3 fallback and TP-aware audio weight loading

URL: https://github.com/sgl-project/sglang/pull/31343
State: closed
Labels: run-ci
Closed at: 2026-07-15T20:26:28Z
Merged at: 2026-07-15T20:26:28Z

## Summary

- **mimo_audio.py**: Remove hard FA3 import (`sgl_kernel.flash_attn.flash_attn_varlen_func`) that blocks module loading on Blackwell (sm103). Replace `AudioEncoderAttention` with `VisionAttention` which auto-selects the correct backend per GPU architecture. Add TP-aware weight remapping in `_remap_audio_tokenizer_state_dict` so the audio encoder loads correctly under tensor parallelism.
- **mimo_vl.py**: Change hardcoded `attn_implementation="flash_attention_3"` to `None`, letting `VisionAttention` auto-select.
- **mimo_v2.py**: Add audio weight remapping in `load_weights` for `q/k/v_proj → qkv_proj` and `out_proj → proj` (skip `input_local_transformer` which uses Qwen2 attention).
- **mimo_v2_asr.py**: Remove `_maybe_override_audio_attn_for_blackwell` workaround, no longer needed.

Without this fix, launching any MiMo-V2 model on GB300 (Blackwell sm103) fails at model import because FA3 is not available, which prevents model class registration and causes `get_model_architecture` to raise `ValueError`.

Closes #30167

## Benchmark on GB300 (Blackwell)

**MiMo-V2.5-Pro** (text-only, 2-node TP8, EAGLE speculative decoding):
- GSM8K (1319 questions, 5-shot): **96.5%** accuracy, 1128 tok/s output throughput

**MiMo-V2.5** (multimodal, single-node TP4):
- MMAU v05.15.25 (1000 samples): **70.3%** overall (sound 78.7%, music 71.0%, speech 61.3%)
