# Add local CuTeDSL BF16 GEMM backend

URL: https://github.com/sgl-project/sglang/pull/30126
State: closed
Labels: quant, jit-kernel
Closed at: 2026-07-04T22:01:12Z
Merged at: 

## Summary

This layers on top of #29865 and removes the dependency on a newer FlashInfer wheel for the BF16 GEMM path by bringing the required CuTeDSL TGV BF16 GEMM kernel into SGLang's JIT kernel tree.

Changes:

- Add a local port of FlashInfer's Blackwell CuTeDSL TGV BF16 GEMM kernel as `sglang.jit_kernel.cutedsl_bf16_gemm`.
- Route `--bf16-gemm-backend flashinfer` through the local JIT kernel on SM100/SM103 instead of importing FlashInfer's unreleased BF16 GEMM symbols.
- Add a registered Blackwell JIT correctness test covering bias and no-bias paths against PyTorch BF16 matmul / linear.
- Add a GLM legacy `deepseek_sparse_attention` config/tokenizer fallback needed to launch `nvidia/GLM-5.2-NVFP4` with the current Transformers strict layer type validation.
