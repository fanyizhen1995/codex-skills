# Route PD server warmup to every DP rank

URL: https://github.com/sgl-project/sglang/pull/30748
State: closed
Labels: run-ci
Closed at: 2026-07-15T07:59:42Z
Merged at: 2026-07-15T07:59:42Z

## Summary

- send one scalar warmup request to each DP rank in disaggregated mode
- route every request explicitly with `routed_dp_rank`
- issue the requests concurrently with `aiohttp` and `asyncio.gather`
- keep `bootstrap_room` as a small unique per-rank identifier
- require successful responses from every DP rank before marking the server ready

## Root cause

The disaggregated server warmup relied on bootstrap-room affinity to spread one Engine API batch across DP workers. In DP-attention deployments, SGLang takes the sequential tokenization path and emits that API batch as individual scheduler dispatches. An explicit load-balancing policy such as `total_tokens` does not use bootstrap-room affinity, and load-snapshot refreshes can overwrite speculative dispatch counters before schedulers publish their new load. The startup warmup therefore does not guarantee that every DP rank executes the dry run.

A rank first touched by real traffic can then pay deferred JIT or initialization cost and introduce rank skew.

## Minimal deterministic reproducer

On the parent with DP8, DP attention, and `total_tokens`, adding `await asyncio.sleep(0.2)` after `self._send_one_request(tokenized_obj)` makes warmup route to `0,0,1,2,3,4,5,6`, missing DP7. With this PR, explicit `routed_dp_rank` routing reaches DP0 through DP7.

## Fix

Build one existing scalar `/generate` request per DP rank with `routed_dp_rank` set explicitly and `bootstrap_room` set to the same rank. Send all requests concurrently through one short-lived `aiohttp.ClientSession` using `asyncio.gather`, while keeping the existing background warmup thread so the HTTP server can serve its own warmup requests. The synchronous warmup entry point creates the client event loop with `asyncio.run`.

The server is marked ready only when every response returns HTTP 200. This avoids changing the public `GenerateReqInput` schema or normal request routing behavior.

## Validation

Validated on two GB300 nodes with DP8/TP8/EP8 and `total_tokens`: all eight routed warmup requests reached DP0 through DP7 successfully, with seven ranks logging their warmup prefill in the same millisecond. Cold-server warmup completed in 264.14 seconds; this startup-only change is not expected to affect steady-state throughput.
