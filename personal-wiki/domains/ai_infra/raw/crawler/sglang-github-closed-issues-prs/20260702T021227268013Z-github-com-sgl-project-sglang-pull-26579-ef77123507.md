# Add SWA-aware KV event metadata

Labels: 
Closed at: 2026-07-01T05:10:46Z
Merged at: 

## Summary
- Extend KV cache `BlockStored` events with optional SWA metadata for `UnifiedRadixCache`: `swa_sliding_window_size` and block-scoped `swa_valid_from`.
- Emit block-local SWA metadata refreshes when Unified SWA cache state changes, including tombstone splits, while preserving full-attention KV placement events.
- Cover wire ordering plus UnifiedRadixCache SWA metadata behavior in unit tests.

Related to #26577.

## Consumer Contract
- `BlockStored` is an idempotent state-refresh event: it means the block is currently stored and carries the latest metadata for that block.
- `BlockRemoved` remains the full-attention KV placement removal signal. SWA-only tombstones are represented by `BlockStored` metadata refreshes, not removals.
- `swa_valid_from` is block/node-scoped. It is set only when that block's own SWA state is missing or tombstoned; a refreshed `None` clears any previous tombstone marker for that block.
- Consumers matching a longer prefix should aggregate metadata along the matched block path and use the deepest non-empty `swa_valid_from` together with `swa_sliding_window_size`.
- Consumers that do not understand the new metadata should be able to ignore the trailing fields and keep existing FULL-only behavior.
