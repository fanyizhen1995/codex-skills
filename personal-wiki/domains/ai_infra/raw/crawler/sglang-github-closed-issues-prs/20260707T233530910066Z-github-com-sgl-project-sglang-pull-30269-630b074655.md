## Summary
- MI355X disagg currently runs the SGLang package baked into the selected Docker image, so workflow_dispatch on a branch can exercise new CI scripts while still serving with image-baked SGLang code.
- Stage the workflow checkout on shared NFS and reinstall its Python `sglang` package inside the prefill, decode, and bench runtime containers before launching the disagg harness.

## Details
- Defaults `SGLANG_USE_CHECKOUT_RUNTIME` to enabled, with `SGLANG_USE_CHECKOUT_RUNTIME=0` as a fallback to use the image-baked package.
- Mounts the staged checkout read-only into runtime containers and copies it to `/tmp/sglang-checkout-runtime` before mutating `python/pyproject.toml`.
- Uses `pip uninstall -y sglang` followed by `pip install --no-deps --no-build-isolation -e .../python`, then asserts that `import sglang` resolves to the checkout copy.
- This intentionally does not rebuild `sgl-kernel`, AITER, MORI, the router, or dependency versions; those still come from the selected image.
