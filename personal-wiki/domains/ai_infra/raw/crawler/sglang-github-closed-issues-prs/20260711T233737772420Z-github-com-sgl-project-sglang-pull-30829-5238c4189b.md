# [eplb] chunk expert-weight P2P on CUDA to prevent NCCL rebalance hang

URL: https://github.com/sgl-project/sglang/pull/30829
State: closed
Labels: documentation, run-ci, run-ci-extra
Closed at: 2026-07-11T04:45:17Z
Merged at: 2026-07-11T04:45:17Z

## Motivation

EPLB rebalance transfers expert weights between ranks with a single `batch_isend_irecv` over all of a layer's experts on CUDA. On large rebalances (hundreds of experts per layer → hundreds of P2P ops in one batch) NCCL can hang GPU-side, so the forward pass driving the rebalance never returns.

ROCm already chunks these P2P ops to avoid the same class of hang. This drops the `_is_hip` gate so the chunking runs on CUDA as well.

## Modifications

- `eplb/expert_location_updater.py`: remove the `_is_hip` special-case so the batched (chunked) P2P submission runs on both CUDA and ROCm.
- Generalize the knob `SGLANG_EPLB_ROCM_P2P_BATCH_CHUNK_SIZE` → `SGLANG_EPLB_P2P_BATCH_CHUNK_SIZE` (default 32), registered as an `EnvIntWithAlias` in `environ.py` with the old ROCm name kept as a deprecated alias. Set the value `>= num_physical_experts` to submit a single batch (previous CUDA behavior).
- Update `docs_new/docs/references/environment_variables.mdx` to the new canonical name.

Behavior-preserving: all ranks iterate the same `num_physical_experts`-based expert-id ranges, so matching send/recv pairs always land in the same batch.
