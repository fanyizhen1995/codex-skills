## Modifications

https://github.com/sgl-project/sglang/pull/29865
Optimization process to collect dynamic autotuning results into static heuristic (and save autotune time + extra compilation time):

1. First collect all the accurate runtime shapes by calling Flashinfer `mm_bf16` for all BF16 linears which will  accurately collect (M,N,K) shape data.
2. Autotune tactics with cold L2 and CUDA graph (M from 1 to 8192) with `flashinfer.testing.bench_gpu_time_with_cupti` and CUDA graph, build a naive heuristic.
3. Then run comprehensive autotune with all tactics (offline), then ask Claude to refine this heuristic based on the empirical data.

For most shapes, <1% difference. It's more conservative, as staying to the default is safer than not.

Default enablement coming later, since it'd benefit from further optimizations.
