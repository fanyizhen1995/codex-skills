# [codex] Add CUDA 12.2 compatibility fallbacks

URL: https://github.com/sgl-project/sglang/pull/29966

## Summary
- Add compatibility fallbacks so current `main` can import, build `sgl-kernel` from source, and run on the CUDA 12.2 / PyTorch 2.1 stack shipped by `nvcr.io/nvidia/pytorch:23.10-py3`.
- Fix old-Torch custom-op ABI issues by converting `std::optional`/tuple returns through a PyTorch-compatible shim and guarding newer dispatcher APIs.
- Gate CUDA components that require newer headers/toolkits: FlashMLA and green-context `spatial_ops` require CUDA >= 12.4, and SM100 MXFP8 sources require CUDA >= 12.8.
- Keep accelerated AOT `sgl_kernel` common ops available on H100/CUDA 12.2 while preserving newer paths when their dependencies are present.

## Notes
- `--sampling-backend pytorch` is used because this clean CUDA 12.2 image does not include the FlashInfer Python package. This PR validates the AOT `sgl_kernel` common-ops path, not FlashInfer sampling.
- `--attention-backend torch_native` is used as the CUDA 12.2-compatible attention path for this environment.
- Full `sgl-kernel/tests/test_sampling.py` collection is blocked in this container by missing `flashinfer.sampling`; the test fixture also expects newer `torch.get_default_device`. Direct AOT op smoke covers the CUDA kernels and old-Torch optional-arg ABI paths relevant to this fix.
