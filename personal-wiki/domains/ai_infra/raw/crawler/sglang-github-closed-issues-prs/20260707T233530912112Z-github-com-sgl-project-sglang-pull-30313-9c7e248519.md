# [AMD] Cap DSV4 Flash max_total_num_tokens

URL: https://github.com/sgl-project/sglang/pull/30313
State: closed
Labels: amd
Closed at: 2026-07-07T07:33:05Z
Merged at: 2026-07-07T07:33:05Z

## Summary
  - Root cause: #28612 changed DSV4 Flash auto-sizing and raised `max_total_num_tokens` far above the previously passing value. On current MI35x images this makes the MORI C4/unified KV RDMA memory region grow to ~5.9-7.9 GiB across DSV4 Flash variants, and MORI fails `RegisterRdmaMemoryRegion` with `errno=22`.
  - This PR keeps the fix surgical: all MI355X DSV4 Flash 1k1k disagg recipes set `runtime.max_total_tokens: 8551168`, and the launcher simply passes that optional recipe field through as `--max-total-tokens`.

## Follow-up
The longer-term fix is likely on the MORI/KV registration side: chunk or otherwise avoid registering a single oversized C4 KV memory region, so capacity does not need a CI recipe cap.
