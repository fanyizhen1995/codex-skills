# Handle NIXL abort notifications

URL: https://github.com/sgl-project/sglang/pull/30685
State: closed
Labels: 
Closed at: 2026-07-09T17:50:06Z
Merged at: 

## Motivation

Decode-side aborts already send best-effort abort notifications to prefill. Mooncake handles those frames, but NIXL prefill still treats them as foreign traffic and can also promote an already-failed room back to an in-progress or successful state from the transfer worker.

## Modifications

- Handle NIXL `ABORT` frames before the bootstrap `GUARD` check.
- Mark known non-success rooms as `KVPoll.Failed`, while preserving successful and unknown rooms.
- Keep NIXL `Failed` status sticky until the sender clears the room, so concurrent transfer-worker completion cannot promote an aborted room back to `Transferring` or `Success`.
- Add focused NIXL unit coverage for known, successful, unknown, malformed, mid-transfer, and sticky-failed status cases.

NIXL intentionally does not send `ABORT_ACK` in this change because there is no decode-side NIXL ACK consumer today; the code leaves a TODO for real ACK/deferred-release semantics if that becomes necessary.
