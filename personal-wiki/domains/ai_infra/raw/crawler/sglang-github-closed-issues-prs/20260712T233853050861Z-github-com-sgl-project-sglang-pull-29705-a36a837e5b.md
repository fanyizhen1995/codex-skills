## Motivation

Our DSpark implementation is based on the baseline from upstream SGLang PR https://github.com/sgl-project/sglang/pull/29538 On top of that baseline, we extended DSpark to support DP attention correctly.

DSpark with DP attention keeps all DP ranks participating in the decode, draft, refine, and verify collectives, even when a local rank has an empty batch. Empty ranks are handled with dummy padded metadata and then trimmed back to their real local batch size after collective-sensitive stages.

For correctness, DSpark refinement must compute logits against the local DP request only. Therefore, when DP attention is enabled, DSpark also enables DP LM head, and the refinement path only gathers vocab shards across the real TP group, not across different DP requests.
