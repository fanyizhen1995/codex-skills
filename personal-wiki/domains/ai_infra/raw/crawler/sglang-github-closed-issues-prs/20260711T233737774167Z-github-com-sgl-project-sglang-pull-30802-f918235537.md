# [refactor] Move MLP collective flags onto ForwardFlags

## Motivation

Decoder layers computed `should_allreduce_fusion`, `use_reduce_scatter`, and `use_flashinfer_trtllm_bypass` once per layer and threaded them through dense MLP, MoE, mamba, and hybrid mixer `forward` signatures. That was pure plumbing: the flags are per-layer control-plane state, not tensor data.

SGLang already has a per-forward tier (`get_forward()` / `ForwardFlags`) for this class of state. Registering these flags there removes kwargs from intermediate modules and matches the existing pattern used for `attn_input_scattered` / `moe_output_buffer`.

## Modifications

- Add graph-visible `ForwardFlags`:
  - `fuse_mlp_allreduce` (was `should_allreduce_fusion`)
  - `mlp_reduce_scatter` (was `use_reduce_scatter`)
  - `flashinfer_trtllm_bypass` (was `use_flashinfer_trtllm_bypass`)
- Decoders publish flags with `get_forward().scoped(...)` around the MLP / MoE / mixer call.
- **Consumer-side skip** (not passed into GEMM):
  - `RowParallelLinear.forward` (and LoRA row-parallel paths) call `should_skip_mlp_all_reduce()`
  - Dense modules just call `self.down_proj(x)` / `out_proj` / `o_proj` without `skip_all_reduce=`
- `should_skip_post_experts_all_reduce(*, is_tp_path)` reads context instead of fusion/RS kwargs.
- Deepseek dual-stream graph op sets `flashinfer_trtllm_bypass` via scoped; custom-op scalar fusion/RS args removed.
- Unit coverage in `test_runtime_context.py`; e2e dumper patch match strings updated.
- Leave `_sglang_needs_allreduce_fusion` on the tensor (cross-layer residual handoff).
