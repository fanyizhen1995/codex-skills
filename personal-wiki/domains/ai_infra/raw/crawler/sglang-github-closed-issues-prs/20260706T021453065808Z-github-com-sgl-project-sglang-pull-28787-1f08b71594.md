# [AMD] Fix RMSNorm batch-invariance on ROCm under deterministic inference

URL: https://github.com/sgl-project/sglang/pull/28787
State: closed
Labels: run-ci
Closed at: 2026-07-05T06:49:06Z
Merged at: 2026-07-05T06:49:06Z

The fused `rms_norm`/`fused_add_rms_norm` kernels used by `RMSNorm.forward_hip` are not batch-invariant: the per-row normalization reduction depends on batch shape, so identical rows can produce slightly different outputs in different batch compositions. Under `--enable-deterministic-inference` this breaks batch-invariance and shows up as logprob "wobble" across batch sizes.

`forward_cuda` already guards against this by falling back to the native path in batch-invariant mode; `forward_hip` did not. Mirror the same guard so ROCm deterministic inference is actually batch-invariant.
