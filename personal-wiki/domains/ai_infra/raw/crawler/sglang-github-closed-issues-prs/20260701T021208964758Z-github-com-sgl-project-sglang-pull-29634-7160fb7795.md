# docker: install dynamo nightly in the dev image for rapid iteration/testing

Labels: 
Closed at: 2026-06-30T05:58:20Z

<!-- codex-pr-description:start -->
Many teams are wanting to try out dynamo router + sglang. This allows them to do it within the dev image (not the release one for now). We install the latest date-stamped `ai-dynamo` nightly in the SGLang development image. This keeps the dev image ready for Dynamo integration testing without enabling a backend extra.

### How This Was Implemented
- Resolve the newest date-stamped nightly from NVIDIA's package index during the image build.
- Reinstall the already-built gateway wheel without dependency resolution so it cannot replace Dynamo's compatible shared dependencies.

<details>
<summary>Walkthrough</summary>

- `docker/Dockerfile`: install the general Dynamo wheel with the other Python tools and preserve the resolved environment when installing the gateway wheel.

</details>

### Validation
- `docker buildx build --check --target framework_final -f docker/Dockerfile .`
- Clean `framework_final` image build with CUDA 13 and `BUILD_TYPE=all`.
- Dynamo frontend to SGLang worker completion on an L40S with `Qwen/Qwen3-0.6B` returned HTTP 200.
- Model gateway to SGLang completion on an L40S returned HTTP 200.
