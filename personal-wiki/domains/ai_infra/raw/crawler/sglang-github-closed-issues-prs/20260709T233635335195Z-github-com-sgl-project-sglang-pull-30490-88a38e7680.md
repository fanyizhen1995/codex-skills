# [refactor] Add the per-forward flags tier: ctx.forward

## Motivation

Per-forward runtime state lived in per-module holders with inconsistent concurrency choices and lifecycles: a thread-local for the multi-stream switch, a bare contextvar for the MoE output buffer, per-forward slots mixed with init-resolved state on the `AttnTpContext` module singleton, and `is_extend_in_batch` as a bare class attribute on the DP buffer wrapper (process-global, sticky, no default).

## Modifications

- **`ctx.forward`** — per-forward flags, each backed by a context variable: attribute reads, `scoped(**kw)` as the one regular write path (transactional, restores on exit and on exception), `set()` for the legacy sticky setters. A new thread sees the defaults, matching both previous holders; TBO interleaves ubatches on one thread (no threading in `batch_overlap/`), so semantics are preserved.
- `do_multi_stream` / `moe_output_buffer` migrate as byte-identical shims (the flashinfer-trtllm runner reads the flag instead of the module contextvar).
- `AttnTpContext`'s per-forward pair (`input_scattered`, `attn_inputs`) moves to the tier; init-resolved fields stay on the class. Safety: input scattering requires `a2a=none` without DP attention while TBO requires the opposite — the two never coexist. `maybe_input_scattered` now restores through `scoped()`, fixing a flag leak when a forward raised (the old in-place swap had no try/finally).
- `is_extend_in_batch`: every writer and reader runs on the single forward thread, so it becomes a forward flag written with the unscoped `set()` — sticky within the thread, with the tier's reset lifecycle.

## Verification

45 runtime-context unit tests (scope nesting / exception restore / thread isolation / attn-slot semantics / sticky semantics); full unit suite name-identical to base ×2; DSV3.2 tp2 smoke (the `maybe_input_scattered` context manager wraps every DeepSeek layer forward).
