# [Feat][GLM5.2] Add DSA Cache Layer Split under Prefill CP

## Summary

Support new features for **GLM5.2 Model**. Add **DSA cache layer-split** for DeepSeek Sparse Attention under prefill context-parallel: the KV/indexer cache layers are sharded across CP ranks (each rank owns a disjoint layer range) instead of every rank holding all layers, cutting per-rank KV memory by ~74% (0.77 → 0.20 GB/rank at 8192 tokens on GLM-5.2-FP8, 78 layers / cp_size=4). Enabled via `--enable-dsa-cache-layer-split` (requires `--enable-prefill-cp  --cp-strategy interleave`).

## Key changes

- **Layer sharding in the memory pool** (`memory_pool.py`): per-rank owned layer ranges, non-owned layers allocated with `num_pages=0`, cross-rank KV/index read via  `_broadcast_tensor_from_owner` over a dedicated CP NCCL communicator.
- **PD transfer support** (`prefill.py`, `mooncake/conn.py`, `common/conn.py`, `base/conn.py`): shard-aware layer-range registration and transfer of owned layers;  decode learns prefill's layer-split state through the bootstrap handshake (`PrefillServerInfo`) and pulls from all CP ranks.
- **CLI / validation** (`server_args.py`): new `--enable-dsa-cache-layer-split` flag, auto-disabled on non-DSA models, decode/non-PD workers, and when prefill-CP interleave isn't set.
- **Two correctness fixes found during e2e validation**:
    - `_broadcast_tensor_from_owner` ran on a default -`disabled` `PyNcclCommunicator` (silent no-op → garbage output); now wrapped in `change_state(enable=True)`.
    - `enable_all_cp_ranks_for_transfer` had no layer-split term, so decode only pulled KV from prefill rank 0 (correct first token, then repetition); now all CP ranks participate on both sender and receiver sides.
   - Unit test for the layer-shard partition.

For better optimization support, it is necessary to rely on the following fix PR.
