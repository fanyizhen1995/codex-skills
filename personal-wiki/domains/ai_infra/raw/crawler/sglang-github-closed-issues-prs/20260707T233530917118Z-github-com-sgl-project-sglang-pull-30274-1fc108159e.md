# [DSA] Fold page-table into fused top-k v2 (decode): drop page_size=1 expansion

## Summary

Integrates the DeepSeek-V4 top-k v2 JIT kernel (`topk_transform_512_v2`, runtime k≤2048, added in #26788) into the shared DeepSeek-V3.2 / GLM DSA sparse-attention path, and removes the page_size=1 page-table overhead it previously required. Python/Triton only — the JIT kernel file is unchanged. Three incremental commits:

**1. Fold the page-table into fused top-k v2 (decode).**
The legacy top-k transform expanded to a `page_size=1` page table. The v2 kernel supports arbitrary page size, so `_try_topk_transform_v2_paged` (in `dsa_topk_backend.py`) feeds it the compact `page_size=64` `real_page_table` (the indexer's own block table) directly. Engaged behind the existing `SGLANG_OPT_USE_TOPK_V2` flag for the decode-shaped PAGED, fp32, 1-row/batch case; everything else (prefill RAGGED, multi-query, non-fp32) falls through to the legacy path unchanged. A one-time INFO log reports engagement for routing observability.

**2. Preprocess the v2 plan once per forward.**
The cluster-threshold plan (`plan_topk_v2`) is now computed once per forward batch into `DSAMetadata.topk_v2_plan` (eager + capture), refreshed in-place with `.copy_()` on CUDA-graph replay, and reused across layers instead of recomputed per top-k call.

**3. Drop the wide `page_size=1` page table in fused decode CUDA graphs.**
The decode CUDA graph statically allocated a `[max_num_tokens, max_ctx_len]` page_size=1 table. With fused top-k (default) it is dead weight — attention reads `topk_indices`, the indexer reads the compact `real_page_table`. The new `dsa_drop_wide_page_table` gate (`cuda ∧ ¬hip ∧ page_size>1 ∧ no-hisparse ∧ no-spec ∧ fuse_topk`) skips the wide allocation and materializes only the compact `real_page_table`, filled straight from `req_to_token` by the fused metadata triton kernels (their `page_table_1` write is made optional). At 160K context this shrinks the static table **64×** (2560 vs 163840 int32 cols/row). Non-fused / hisparse / HIP-indexer / spec (MTP precompute, target-verify, draft-extend) paths keep the wide table and are untouched.

## Verification

- **B200 unit**: the three fused metadata triton kernels produce **byte-identical** `real_page_table` with `page_table_1=None` vs the wide-table path.
- **B200 wiring**: the drop-path decode-graph `real_page_table` is byte-identical (same shape + values) to the legacy derive-from-wide result, verified up to **160K context**.
- **v2 correctness**: v2 selection ≡ v1 ≡ `torch.topk` and the fold yields identical physical KV slots to `page_size=1`, through the real `DSATopKBackend.topk_transform`, up to 1M context; CUDA-graph capture/replay safe.
- **E2E**: DeepSeek-V3.2 GSM8K (TP8) — validated via the registered 8-gpu-h200 test.

## Compatibility

GLM DSA (`index_topk`) shares this exact code path, so it is covered without extra code. All new behavior is gated; default configs on non-DSA / non-CUDA / spec paths are byte-unchanged.
