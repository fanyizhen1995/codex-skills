# Fix processor config loading for object-storage model paths

Closed at: 2026-07-15T07:21:32Z

## Summary

- resolve `model_name` through the object-storage cache before loading its config
- add a CPU regression test for a local tokenizer paired with an object-storage model path

## Why

`get_processor()` already resolves `tokenizer_name`, but when a separate `model_name` is supplied, `AutoConfig` loads from that argument instead. For RunAI-backed S3/GCS/Azure models, metadata has already been mirrored locally, so passing the unresolved URI to `AutoConfig` bypasses the prepared cache path and can fail processor initialization.
