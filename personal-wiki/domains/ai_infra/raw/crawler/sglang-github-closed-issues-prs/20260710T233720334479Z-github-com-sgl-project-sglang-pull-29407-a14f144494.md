# Localize cur_batch field in Scheduler to avoid field-based state access

URL: https://github.com/sgl-project/sglang/pull/29407
State: closed
Labels: apple-silicon
Closed at: 2026-07-10T00:55:07Z
Merged at: 2026-07-10T00:55:07Z

`self.cur_batch` was a scheduler-wide attribute written by every event loop (normal, overlap, PP, disagg prefill/decode, MLX) but its only cross-thread reader is the watchdog, which uses it purely as a debugging signal (`is_active` / `dump_info`). All in-loop consumers can read the batch from a local instead.

This PR renames the field to `cur_batch_for_debug` to make the watchdog its sole legitimate user, and stops every other site from depending on it:

- `launch_batch_sample_if_needed` takes the current batch as a parameter instead of reading `self.cur_batch`; all three callers pass their local `batch`.
- The PP loops bind a local `cur_batch` and only mirror it into `cur_batch_for_debug` for the watchdog; `_pp_launch_batch` takes that local `cur_batch` as a parameter.

Pure rename + local threading; no behavior change. The event loops still write `cur_batch_for_debug` every iteration, so the watchdog signal is unchanged. Tests that poke the attribute directly are updated to the new name.

## Equivalence audit

Verified behavior-preserving by an independent read-only Codex audit (verdict: PROVEN). It confirmed the rename mapping is complete — every former runtime write of `self.cur_batch` now writes `self.cur_batch_for_debug` at the same point (so the watchdog observes the same value at the same times), the watchdog reads map to the new name with no remaining `self.cur_batch` residual, and every former non-watchdog read is preserved as a same-object local/parameter. For `_pp_launch_batch`, all three call sites pass exactly the object `self.mbs[mb_id]` would have returned.
