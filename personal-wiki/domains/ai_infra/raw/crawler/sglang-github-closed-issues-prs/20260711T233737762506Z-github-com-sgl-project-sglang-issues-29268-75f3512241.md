# [Bug] DeepSeek multithreaded weight loader deadlocks on slow/networked storage (DP-attention, multiple ranks per node): CUDA driver rwlock held across mmap page-fault

URL: https://github.com/sgl-project/sglang/issues/29268
State: closed
Labels: 
Closed at: 2026-07-11T06:58:43Z
Merged at: 

## Summary

When loading a large DeepSeek-family checkpoint with **DP-attention and multiple ranks per node** from **slow or cold/networked storage** (NFS/Lustre), the multithreaded safetensors weight loader serializes/deadlocks badly enough that one or more ranks miss the initialization barrier and the scheduler dies:

```
RuntimeError: Rank 0 scheduler died during initialization (exit code: -3).
```

On **fast local NVMe (or warm page cache)** the exact same configuration loads fine — which is why this is intermittent and cluster/storage-dependent. This is the same failure class as #7905 (Slow Model Loading on GB200: DeepSeek-R1-FP4, 30+ min, ~50% timeout), and is the load-time manifestation of the DP-attention redundant-mmap behavior that the loader's own `_prefetch_all_checkpoints` docstring already documents.

## Symptoms

- Multi-rank-per-node load (DP-attention + TP, e.g. 4 ranks/node) from networked/cold storage **stalls**; ranks fail to reach the init/NCCL barrier within `watchdog_timeout` (default 300 s) → `scheduler died (exit code -3)`.
- **Warm/local NVMe**: loads fine.
- **Single-threaded** loading: never deadlocks (just slow — and on cold NFS still exceeds the 300 s watchdog).
- `py-spy dump --native` of the stuck ranks shows worker threads blocked in the CUDA driver's `pthread_rwlock_wrlock` inside `cuMemcpyHtoDAsync`, while the source pages are being faulted in (`folio_wait_bit_common`, NFS read) — i.e. the page fault on the pageable mmap'd source happens **while the global driver lock is held**.

## Diagnosis (root cause)

The DeepSeek weight loader (`deepseek_common.do_load_weights`) submits up to 32 concurrent weight copies via `ThreadPoolExecutor()` (`maybe_executor_submit` in `srt/model_loader/utils.py`). Each worker performs an H2D copy from a **pageable, mmap-backed** CPU tensor (`safe_open(...).get_tensor()`, the default since `weight_loader_disable_mmap=False`).

`cuMemcpyHtoDAsync` (libcuda) holds a **global, process-wide driver `rwlock`** for the duration of the staged copy. When the CPU source is a pageable mmap, the copy **page-faults on the source while that lock is held**. On slow storage the fault blocks on I/O (`folio_wait_bit_common`) — so the global driver lock is held across slow I/O, and **every other loader thread (indeed every CUDA driver call in the process) serializes behind it**. With N ranks/node each independently faulting the same shards, contention compounds. Accumulated stalls exceed the init barrier / `watchdog_timeout`, and a rank is declared dead.

Key property: the multithreading provides **no** parallelism in this path — the threads serialize on the driver lock — but it converts slow-storage latency into a hard failure (barrier timeout) instead of merely a slow load.

## Fixes evaluated

| Strategy | Mechanism | Deadlock-safe | Works on NVFP4 |
|---|---|---|---|
| **Patch: materialize source in worker** | `.clone()` the mmap tensor in the worker → fault happens off the driver lock | **Yes (unconditional)** | Yes |
| `--weight-loader-prefetch-checkpoints` | bg thread warms page cache (seq 16 MB reads), shards split across local ranks | Yes, **if prefetch stays ahead** (RAM ≥ model) | Yes |
| `--weight-loader-disable-mmap` | eager `safetensors.torch.load(read())` | (n/a) | **No — `KeyError: 'F8_E8M0'`** |
| single-thread (`max_workers=1`) | no concurrency, no lock contention | Yes | Yes (but slow; >300 s watchdog on cold NFS) |
| stage weights to local NVMe | avoids the slow fault entirely | Yes | Yes |

### Possible patch

Materialize the mmap source resident **inside the worker thread** before the copy, so the page fault happens in the worker (in parallel, **no driver lock held**); the subsequent H2D copy then reads already-resident memory and holds the lock only briefly:

```python
# srt/model_loader/utils.py, inside maybe_executor_submit
def _run_materialized(func, func_args, func_kwargs):
    def _mat(x):
        if isinstance(x, torch.Tensor) and x.device.type == "cpu":
            return x.clone()          # force the page-in here, off the CUDA driver lock
        return x
    return func(*tuple(_mat(a) for a in func_args),
                **{k: _mat(v) for k, v in (func_kwargs or {}).items()})

# replace the submit:
futures.append(executor.submit(_run_materialized, func, func_args, func_kwargs))
```

Cost: a transient per-weight clone, measured at **~1.77 GB/rank peak (~7 GB/node for 4 ranks)**, freed as each copy completes. Eliminates the deadlock unconditionally because the fault is never under the driver lock. (Note: `--weight-loader-prefetch-checkpoints` is preferable when node RAM ≥ model size — see below — since it needs no patch and its memory cost is reclaimable page cache; the patch is the unconditional fallback, e.g. when the model does not fit in node RAM.)

### Notes on the existing flags

- **`--weight-loader-prefetch-checkpoints`** is the best *existing* mitigation when **node RAM ≥ model size**: it warms page cache ahead of the loader and splits shards across the local ranks so the checkpoint crosses the network **once** (not N×). It neutralizes the slow fault and was fastest on cold NFS in our tests, with only reclaimable page-cache cost. Caveat: it makes the fault *fast* (cache hit) rather than moving it out from under the lock, so it relies on prefetch staying ahead; if `model > node RAM` (or storage is slower than prefetch can read), a cold fault under the lock can still occur.
- **`--weight-loader-disable-mmap`** does **not** work for NVFP4 checkpoints: it routes through `safetensors.torch.load()` which raises `KeyError: 'F8_E8M0'` (the eager deserializer does not know the E8M0 microscaling-scale dtype). It crashes during init regardless of storage.

## Benchmarks

Single GB300 node, TP4 / DP4 / EP4, DP-attention, DeepSeek-V4 NVFP4 (806 GB, 64 shards). Per-rank weight-load time from the loader's own `Load weight end. elapsed=`. Cold NFS = page cache dropped (`POSIX_FADV_DONTNEED`) before each load.

**Load time (per-rank):**

| Strategy | local NVMe (warm) | cold NFS |
|---|---|---|
| baseline multithread (current default) | 71 s | **deadlock → barrier timeout** |
| single-thread | 59 s | 537 s (> 300 s watchdog) |
| **materialize patch** | **43 s** | 264 s |
| `--weight-loader-disable-mmap` | crash (`F8_E8M0`) | crash (`F8_E8M0`) |
| **`--weight-loader-prefetch-checkpoints` (8 thr)** | 46 s | **58 s** |

**Peak host memory:**

| Strategy | anonymous RSS (4 ranks, non-reclaimable) | extra cost |
|---|---|---|
| baseline / single-thread | ~25–32 GB | mmap fault working set |
| materialize patch | ~14–24 GB | **+~1.77 GB/rank** transient clones |
| prefetch | ~23 GB (≈ baseline) | **reclaimable page cache only** |
