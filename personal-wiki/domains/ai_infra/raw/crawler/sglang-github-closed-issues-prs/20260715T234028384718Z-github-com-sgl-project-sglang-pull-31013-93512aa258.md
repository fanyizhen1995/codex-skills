# [Spec] Single-source `num_tokens_per_req` derivation and access

URL: https://github.com/sgl-project/sglang/pull/31013
State: closed
Labels: blackwell, run-ci
Closed at: 2026-07-15T01:41:09Z
Merged at: 2026-07-15T01:41:09Z

Follows up on #30977 / #30998. Converges the speculative per-request token width to one dynamic access point and one static derivation point, with no behavior change:

- **Dynamic (per-forward)**: declare `num_tokens_per_req` / `num_tokens_for_logprob_per_req` on the `SpecInput` base and give `get_spec_adjust_token_coefficient` a base implementation; five of the six per-subclass overrides collapse into it (verified value-equivalent per class, including every post-construction writer of the fields). Consumers stop re-deriving: two `num_tokens // bs` inversions and three `getattr`-with-fallback reads become direct field reads.
- **Kept override**: `EagleVerifyInput.get_spec_adjust_token_coefficient` stays on `draft_token_num` — the v2 worker re-stamps `num_tokens_per_req = num_steps + 1` on every `verify()`, which diverges from the real verify width for `topk > 1` trees, so the DP-attention global-token scaling must not read the stamped field (see the inline comment).
- **Static (capture-time)**: add `spec_utils.resolve_num_tokens_per_req(phase=...)` (phases typed as `Literal`; draft phases are EAGLE-family-only for now) as the single derivation of draft-decode / draft-extend / target-verify widths; graph runners, `ModelRunner.decode_num_tokens_per_req`, the eager/base runners, and the fa3 draft-worker adjustment all route through it. `get_num_tokens_per_req_for_target_verify` now has exactly one caller.
